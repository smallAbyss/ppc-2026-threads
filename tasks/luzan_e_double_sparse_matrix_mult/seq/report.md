В данной работе рассматривается задача умножения разреженных матриц, представленных в формате CCS (Compressed Column Storage).

Последовательная версия алгоритма используется как базовая реализация для:

## 2. Постановка задачи

### Входные данные

Даны две разреженные матрицы вещественных чисел:

$$  
A \in \mathbb{R}^{M \times K}, \quad  
B \in \mathbb{R}^{K \times N}  
$$

Матрицы представлены в формате CCS.

Структура `SparseMatrix` содержит:

- `value` - значения ненулевых элементов;
- `row` - индексы строк соответствующих элементов;
- `col_index` - границы диапазонов столбцов;
- `rows`, `cols` - размеры матрицы.

### Выходные данные

Результирующая матрица:
$$  
C = A \times B  
$$
также сохраняется в формате CCS.

### Ограничения

Умножение возможно только при выполнении условия:
$$  
A.cols = B.rows  
$$
При нарушении этого ограничения вычисление не выполняется.
Для подавления накопления ошибок округления используется порог:

$$  
kEPS = 10^{-8}  
$$

Элементы с модулем меньше данного значения считаются нулевыми и не сохраняются в результат.

### Граничные случаи

Обрабатываются следующие ситуации:

- матрицы из одного элемента
- столбцы без ненулевых элементов;
- частично нулевые промежуточные результаты.

## 3. Базовый алгоритм

Алгоритм вычисляет результат по столбцам результирующей матрицы.
Для каждого столбца матрицы (B):

1. создаётся временный плотный аккумулятор `tmp_col`;
2. перебираются ненулевые элементы текущего столбца (B);
3. для каждого элемента выбирается соответствующий столбец матрицы (A);
4. произведения накапливаются в `tmp_col`;
5. ненулевые значения сохраняются в результирующую CCS-структуру.

### Асимптотика

#### Временная сложность

$$  
O(cols_B \times rows_A + nnz_B \times nnz_per_col_A)  
$$

где:

- `nnz_B` - количество ненулевых элементов матрицы (B);
- `nnz_per_col_A` - среднее количество ненулевых элементов в столбцах матрицы (A).

#### Пространственная сложность

$$  
O(rows_A)  
$$
Дополнительная память расходуется только на временный аккумулятор текущего столбца.

### Инварианты корректности

Во время вычислений сохраняются следующие свойства:

- каждый столбец результата вычисляется независимо;
- элементы внутри столбца сохраняются в порядке строк;
- в результирующую матрицу не записываются значения меньше `kEPS`;
- структура `col_index` всегда содержит корректные границы столбцов.

## 4. Детали реализации

### Структура файлов

- `common/include/common.hpp` - структура `SparseMatrix` и реализация умножения;
- `seq/include/ops_seq.hpp` - объявление sequential backend;
- `seq/src/ops_seq.cpp` - реализация этапов выполнения задачи.

### ValidationImpl

Функция проверяет корректность размеров входных матриц.

```cpp
bool ValidationImpl() {
    return matr_a.cols == matr_b.rows;
}
```

При несовместимых размерах вычисления не запускаются.

### PreProcessingImpl

На этапе предварительной обработки подготавливаются ссылки на входные и выходные данные, необходимые для вычислений.
Дополнительных преобразований данных последовательная версия не требует.

### RunImpl

Основной этап вычислений:

```cpp
GetOutput() = matr_a * matr_b;
```

Умножение выполняется через перегруженный оператор `operator*`.

### PostProcessingImpl

После завершения вычислений результирующая CCS-структура передаётся вызывающему коду без дополнительных преобразований.

## 5. Проверка корректности

Для проверки корректности использовались файловые тесты на базе Google Test.

### Метод проверки

Каждый тест:

1. считывает две входные матрицы из файла;
2. запускает умножение;
3. сравнивает результат с эталонной CCS-структурой, загруженной из файла.

### Набор тестов

Используются 6 файловых тестов:

- `test_1.txt` - `test_6.txt`;
- `ans_test_1.txt` - `ans_test_6.txt`.

Все тесты успешно пройдены.

### Примеры тестов

#### Пример 1

Вход:
$$  
A_{2\times3}, \quad B_{3\times5}  
$$
Результат:

$$  
C_{2\times5}  
$$
Проверяется корректность базового перемножения.

#### Пример 2

Вход:

$$  
A_{1\times1}, \quad B_{1\times1}  
$$
Проверяется обработка минимального размера матриц.

### Анализ времени выполнения

Наиболее затратным фрагментом алгоритма является внутренний цикл накопления произведений в плотный временный аккумулятор.
Основная вычислительная нагрузка связана с:

- последовательным обходом ненулевых элементов;
- накоплением промежуточных сумм;
- обработкой CCS-структур.

При этом использование плотного аккумулятора позволяет избежать дорогостоящих поисков элементов внутри разреженной структуры.
